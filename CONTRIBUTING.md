# Contributing to FlowTrack

Thanks for your interest in improving FlowTrack.

## Prerequisites

- Node.js 20+
- npm 10+

## Local setup

```bash
npm install
```

## Build and checks

Use these commands before opening a pull request:

```bash
npm run build
```

If you add or update tests, run the relevant test command(s) as part of your change.

## Development guidelines

- Keep changes focused and small when possible.
- Follow the existing TypeScript style and project structure.
- Update `README.md` when behavior, configuration, or public usage changes.
- Add or update tests for behavior changes.

## Pull request process

1. Fork the repository and create a feature branch from `main`.
2. Make your changes and verify build/tests locally.
3. Open a pull request with:
   - clear summary of what changed,
   - motivation or linked issue,
   - testing notes (what you ran).
4. Address review feedback and keep the branch up to date.

## Reporting issues

Please open issues at:

- https://github.com/ljuba-dev/flowtrack/issues

Include environment details, expected behavior, actual behavior, and minimal reproduction steps.
